Prework Study Guide

HTML

  • HTML is used to create the structure of a webpage.
  • HTML elements work together to provide a framework that makes it easy for a user to navigate.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS defines the colors, aesthetics, visual layout, and organisation of a document written in HTML.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • When using an external style sheet, we MUST link the file using: rel="stylesheet" href="./assets/style.css, between a link element in the head of the HTML.
  • The selector defines which elements the rule will apply to e.g. p {declaration}, when refering to the properties of the text in a paragraph.
  • The declaration contains the property we want to apply and the value of the property e.g. color: blue;
  • A period preceding a property designates this as a class selector. Any HTML element assigned to the class will gain these property assignments from the CSS.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside the element.
  • Anything written between /* and */ is a CSS comment and will be ignored by the browser

GIT

  • Version control system
  • Doesn't use the same as bash commands, always begins with "git"
  • git status: what branch we are on
  • git checkout -b feature/[name of branch]: creates a new branch and switches to it

Git Flow


  1. git add -A: to stage files after any changes are saved
  2. git commit -m "insert commit message": after file is saved
  3. git pull origin main: to make sure we have the latest commits and changes
  4. git push origin feature/[name of branch]: push your commits back to Git Hub

Clone Repository to Local Computer


    Copy SSH value from "Code"
    Open terminal
    cd into the folder you want your repo stored
    git clone [paste copied SSH]
    cd [name of repo]
    git remote -v to check the connection between the local and remote repo
    code .
    File will be opened with VS Code
