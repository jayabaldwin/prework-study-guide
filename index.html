HTML

  • HTML is used to create the structure of a webpage.
  • HTML elements work together to provide a framework that makes it easy for a user to navigate.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS defines the colors, aesthetics, visual layout, and organisation of a document written in HTML.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • External style.css files are linked within the head of the HTML using: rel="stylesheet" href="./assets/style.css, betweem a link tag.
  • The selector defines which elements the rule will apply to e.g. p {declaration}, when refering to the properties of the text in a paragraph.
  • The declaration contains the property we want to apply and the value of the property e.g. color: blue;
  • A period preceding a property designates this as a class selector. Any HTML element assigned to the class will gain these property assignments from the CSS.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside the element.
  • Anything written between /* and */ is a CSS comment and will be ignored by the browser

GIT

  • Version control system
  • Doesn't use the same as bash commands, always begins with "git"
  • git status: what branch we are on
  • git checkout -b feature/[name of branch]: creates a new branch and switches to it

Git Flow


  1. git add -A: to stage files after any changes are saved
  2. git commit -m "insert commit message": after file is saved
  3. git pull origin main: to make sure we have the latest commits and changes
  4. git push origin feature/[name of branch]: push your commits back to Git Hub

Clone Repository to Local Computer


  1. Copy SSH value from "Code"
  2. Open terminal
  3. cd into the folder you want your repo stored
  4. git clone [paste copied SSH]
  5. cd [name of repo]
  6. git remote -v to check the connection between the local and remote repo
  7. code .
  8. File will be opened with VS Code

JavaScript

  • JavaScript (JS) allows users to interact with websites using effects, interactivity, data communication etc.
  • Connect script.js files with a [script] tag under the footer, within the body element.
  • A variable is a named container that allows us to store data in our code [var].
  • A string is a sequence of text, the value is enclosed in quote marks. To access the string refer to the variable name.
  • .log() will output whatever is inside the parentheses to the console using [console.log()]
  • Control flow is the order in which a computer executes code in a script.
  • Using conditional statements allows us to interrupt control flow.
  • [if] statements: provide one or more conditions for the computer to evaluate. If one condition is truthy, the computer will execute the code that we provide between curly brackets.
  • An array is a single variable used to hold a group of related data and can store multiple values in a single reference.
  • Indexing is used to work with individual data inside an array, following the variable name, beginning at [0].
  • [for] loops contain 3 important statements. Starting point e.g. var x = 0; then the condition, x < the variable times the length of the array and the final statement x++ to express the pattern (x+1 indicates increasing indicies).
  • for(x = 0; x < variable.length; x++) {// code block}]
  • A function is a set of instructions that tells the computer how to perform a certain task. It must be called by name in order to execute it [function name()].
